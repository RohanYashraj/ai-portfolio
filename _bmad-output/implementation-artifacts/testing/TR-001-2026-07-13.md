## Environment caveat (matters for interpreting evidence)

The browser-automation pane throttles rAF/compositing: count-up stats freeze at first frame, native smooth scroll freezes mid-flight, and wheel-scrolling the animation-heavy home page can hang the pane renderer (30s timeout, blank captures afterward). **These are automation-environment artifacts, not page defects**: SSR HTML carries final stat values (7+/1/1 confirmed via curl), animation math self-heals on any real frame (confirmed — forcing one frame completed the count-up), and all affected behaviors were verified interactively during the [D] build of 1.1/1.2. /archive (no animations by design) never exhibits any of it. Flagged as OBS-01 with human-pass items below.

## Issues Found

### ISS-001 (MEDIUM, fixed): No social sharing metadata (og:/twitter:) on any page
LinkedIn is a primary entry channel (spec 1.1 entry points; Elena arrives via LinkedIn/peer links) — links shared there render with no preview card. **Expected:** og:title/description (+image when available), twitter:card. **Fix:** `openGraph` + `twitter` in metadata exports, `metadataBase` in root layout.

### ISS-002 (MEDIUM, fixed): robots.txt and sitemap.xml 404
"Own the AI Actuary phrase" is the engine goal; crawl guidance is baseline SEO. **Fix:** `app/robots.ts` + `app/sitemap.ts` (Next metadata routes), sitemap listing `/` and `/archive` (rooms added as they're built).

### ISS-003 (LOW, fixed): No canonical URL
Folded into ISS-001 fix via `metadataBase` + `alternates.canonical`.

### ISS-004 (LOW, fixed): /archive heading hierarchy skips h1→h3
Entry titles are `h3` with no `h2` on the page. **Fix:** year subdividers become `h2` (they are genuine section headings: "2026", "2025", …), preserving h1→h2→h3.

### OBS-01 (informational, no action): animation suspension under throttled renderers
Count-up/type-in/smooth-scroll are rAF/compositor-driven and pause in throttled contexts (automation panes, hidden tabs); they self-heal on the next real frame and SSR carries final values. No launch risk identified. Optional hardening (not required): finish count-up via `visibilitychange`.
