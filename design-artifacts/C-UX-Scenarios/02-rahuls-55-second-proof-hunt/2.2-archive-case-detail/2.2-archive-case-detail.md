## Overview

**Page Purpose:** Prove depth and judgment. One template serving projects, papers, and talks with type-specific fields: context (problem) → approach (process) → results (dated, concrete) → artifacts (repo, paper, slides, recording). The tap must repay itself within seconds.

**User Situation:** Rahul tapped the strongest-looking entry; ~30 seconds of budget left. Headline result must be visible without scrolling. This is where "commodity candidate" dies or survives.

**Success Criteria:** Headline result above the fold; scenario success = Rahul reads to results with conviction formed ✓. Artifact links verifiable in one tap.

**Entry Points:**
- Archive entry tap (primary) · Selected Works placard · Speaking & Writing entry · docent deep-link · external shares/citations (this page gets direct traffic)

**Exit Points:**
- Artifact links (new tab) · related entries footer → next case · back to Archive (filter preserved) · DocentPanel

## Layout Structure

```
+--------------------------------------+
| SiteHeader                           |
+--------------------------------------+
| ← The Archive          (breadcrumb)  |
| 2026 · Project           (metadata)  |
| CLAIMS TRIAGE WITH LLMS    (title)   |
| Cut manual review 40% at a top-5     |
| insurer          (headline result)   |  ← above the fold, always
+--------------------------------------+
| ARTIFACTS: [Repo ↗] [Slides ↗]       |
+--------------------------------------+
| THE PLACARD (structured body):       |
|  Context   — the problem, 2-3 sent.  |
|  Approach  — process & judgment      |
|  Results   — concrete, dated         |
+--------------------------------------+
| RELATED: | placard | placard |       |
+--------------------------------------+
                          [Docent ✳]
```

Type variations (same skeleton): **Project** — context/approach/results + repo/demo. **Paper** — abstract-as-context, method, findings + DOI/PDF. **Talk** — event context (venue, audience), topic, takeaways + slides/recording.

## Technical Notes

- Artifact links checked by a scheduled link-health job (a dead repo link = the staleness fear realized)
