## Overview

**Page Purpose:** Everything in one filterable room — all projects, papers, talks as dated, typed entries. Currency visible at a glance (newest first, dates prominent); depth exactly one tap away.

**User Situation:** Rahul, between meetings on his phone, 55-second budget ticking. Hunting artifacts, allergic to staleness. Ananya browses here too, on slower evenings.

**Success Criteria:** Time-to-first-relevant-entry <10s; filter use requires zero learning; visible dates answer the staleness question before it's asked.

**Entry Points:**
- Room nav · "See everything" cue from Selected Works · Home stats (papers → filtered) · docent deep-link · external links to filtered views

**Exit Points:**
- Entry tap → Case Detail (primary) · filter stays on page · DocentPanel

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | All types present | Full list, "All" active | Filter, tap |
| Filtered | Chip selected | Narrowed list; count shown ("8 papers") | Clear, tap |
| Filtered-empty | Type has no entries | Chip disabled up-front (never an empty result screen) | Other filters |
| Reduced-motion | prefers-reduced-motion | No list transitions | All |
| Deep-linked | `?type=` in URL | Pre-filtered on load | All |

---

## Technical Notes

- List renders statically (SSG) with client-side filter enhancement — must be fully readable pre-hydration on mid-range Android
- Disabled-chip rule enforces "no dead sections": a type with zero entries never presents an empty room
- Entry order: date desc; CMS pin override available but discouraged here (currency is the message)
